# Petstore Postman Collection

## Як запускати

1. Відкрити Postman
2. Імпортувати колекцію (файл):
   - Petstore.postman_collection.json
3. Натиснути **Run** у Postman → запускати колекцію.

## Змінні колекції

- `{{base_url}}` — базовий url для запитів
  
Наступні змінні генерується динамічно у Pre-request Script запитів, і видаляються після виконання останнього запиту
- `{{petId}}`
- `{{petName}}`
- `{{updatedPetName}}`
- `{{newStatus}}`


## Тест-кейси для ендпоінта add a new pet to the store POST /pet

1. Користувач може додати нову тварину заповнивши всі обов’язкові та додаткові поля валідними даними.
   
   **Очікуваний результат:** Тваринка створена успішно, статус відповіді 200, у відповіді всі поля співпадають із полями в запиті. 

2. Користувач може додати нову тварину заповнивши лише обов’язкові поля валідними даними.
   **Очікуваний результат:** Тваринка створена успішно, статус відповіді 200, у відповіді всі поля співпадають із полями в запиті, додаткові поля порожні або дефолтні. 

3. Користувач отримує помилку, якщо не передасть в body обов'язкове поле "name" в запит створення нової тварини.
   **Очікуваний результат:** API повертає помилку 400 Bad Request із повідомленням про відсутнє поле "name". 

4. Користувач отримує помилку, якщо передасть в body обов'язкове поле "name" зі значенням порожньої string або null. ("" / null)
   **Очікуваний результат:** API повертає помилку 400 Bad Request із повідомленням, що значення не може бути порожнім. 

5. Користувач отримує помилку, якщо не передасть обов'язкове поле "photoUrls" в запит створення нової тварини.
   **Очікуваний результат:** API повертає помилку 400 Bad Request із повідомленням про відсутнє поле "photoUrls". 

6.  Користувач отримує помилку, якщо передасть в body обов'язкове поле "photoUrls" зі значення порожнього масиву, або null. ([] / null)
   **Очікуваний результат:** API повертає помилку 400 Bad Request із повідомленням, що значення не може бути порожнім.

7. Користувач не може створити тварину вказавши в полі "id" значення, уже існуюче для іншої тварини в магазині. Значення "id" має бути унікальним.
   **Очікуваний результат:** API повертає помилку 400 Bad Request із повідомленням, що значення в полі id має бути унікальним. 

8. Користувач не може створити тварину вказавши в полі "id" значення з некоректним типом даних (наприклад string, object, array).
   **Очікуваний результат:** API повертає помилку 400 Bad Request із повідомленням, що значення в полі id має некоректний тип даних.

9. Користувач отримує помилку, вказавши в полі "status" не валідне значення (відмінне від [ available, pending, sold ])
    **Очікуваний результат:** API повертає помилку 400 Bad Request із повідомленням, що значення в полі status не валідне.

10. Користувач може додати тварину в магазин, вказавши більше 1 тега.
    **Очікуваний результат:** Тваринка створена успішно, статус код-200, усі передані теги у відповіді присутні.
